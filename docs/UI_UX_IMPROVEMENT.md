# Clork UI/UX 개선 제안서

## 현재 UI 분석

### 문제점
1. **글자 크기가 작음**
   - base: 15px (HTML root)
   - text-sm: 13px (대부분의 컴포넌트)
   - text-xs: 12px (작은 텍스트들)

2. **컴포넌트 크기가 작음**
   - 버튼: px-4 py-2
   - 전반적으로 컴팩트한 UI
   - 터치 타겟이 작음

3. **색상이 단조로움**
   - 주색상: Indigo (primary)
   - 회색 계열 위주
   - 대비가 약함

4. **레이아웃이 빽빽함**
   - 여백이 부족
   - 시각적 계층 구조가 불명확

## UI/UX 개선 제안

### 1. Bento Box 스타일 (애플 스타일의 모던 UI)
- **특징**: 크고 시원한 카드 기반 레이아웃, 넉넉한 여백, 부드러운 그림자
- **글자 크기**: base 16-18px, 주요 텍스트 14-16px
- **장점**: 가독성 극대화, 시각적 계층 구조 명확, 터치 친화적
- **색상**: 밝은 파스텔톤 액센트 색상 + 깊은 그림자로 depth 표현
- **참고**: Linear, Raycast, Craft 앱 스타일

### 2. Glass Morphism 스타일 (반투명 유리 효과)
- **특징**: 반투명 배경, backdrop blur 효과, 얇은 보더
- **글자 크기**: base 16px, 더 굵은 폰트 웨이트
- **장점**: 모던하고 세련된 느낌, 레이어 구분 명확
- **색상**: 생동감 있는 그라데이션 + 반투명 오버레이
- **참고**: Windows 11, macOS Big Sur 스타일

### 3. Notion 스타일 (미니멀하고 기능적)
- **특징**: 깔끔한 타이포그래피, 여백 활용, 아이콘 적극 활용
- **글자 크기**: base 16px, 헤더는 더 크게 (18-24px)
- **장점**: 콘텐츠 중심, 깔끔한 정보 구조
- **색상**: 제한된 색상 팔레트, 회색 계열 + 1-2개 포인트 색상
- **참고**: Notion, Obsidian 스타일

### 4. GitHub/Linear 하이브리드 (개발자 친화적)
- **특징**: 어두운 테마 중심, 네온 액센트, 코드 에디터 느낌
- **글자 크기**: 모노스페이스 폰트 14-16px, UI 텍스트 15-16px
- **장점**: 개발 도구와 일관된 경험, 눈의 피로 감소
- **색상**: 어두운 배경 + 밝은 네온 색상 (cyan, magenta, lime)
- **참고**: GitHub Copilot, Linear, Vercel Dashboard

### 5. Material You (구글의 최신 디자인)
- **특징**: 큰 둥근 모서리, 다이나믹 색상, 부드러운 애니메이션
- **글자 크기**: base 16px, 다양한 폰트 스케일
- **장점**: 친근하고 부드러운 느낌, 접근성 우수
- **색상**: 사용자 맞춤 색상 테마, 톤 온 톤 배색
- **참고**: Google Workspace, Android 13+

## 추천 방향: Bento Box + Glass Morphism 혼합

### 시각적 개선
1. **타이포그래피**
   - 글자 크기 전체적으로 20-25% 증가
   - base font-size: 16px → 18px
   - 주요 텍스트: 14px → 16px
   - 작은 텍스트: 12px → 14px
   - 행간(line-height) 1.5 → 1.6-1.7

2. **스페이싱**
   - 카드 패딩: 현재의 1.5-2배
   - 섹션 간격: 24px → 32-40px
   - 버튼 패딩: px-6 py-3 (현재 px-4 py-2)

3. **둥근 모서리**
   - 카드: border-radius 16-24px
   - 버튼: border-radius 12-16px
   - 입력 필드: border-radius 12px

4. **그림자와 효과**
   - 부드러운 다층 그림자
   - hover 시 elevation 변화
   - 포커스 상태 강조

### 색상 개선
1. **메인 색상**
   - Indigo → 더 생동감 있는 보라색/청록색
   - 그라데이션 배경 추가
   - 다크 모드에서 네온 액센트

2. **상태별 색상**
   - 성공: 밝은 초록색
   - 실행 중: 생동감 있는 파란색
   - 경고: 선명한 주황색
   - 오류: 부드러운 빨간색

### 사용성 개선
1. **인터랙션**
   - 드래그 앤 드롭 시각적 피드백 강화
   - 로딩/전환 애니메이션 추가
   - 마이크로 인터랙션 추가

2. **접근성**
   - 터치 타겟 크기 증가 (최소 44px)
   - 키보드 단축키 추가
   - 고대비 모드 지원

3. **반응형**
   - 모바일 친화적 레이아웃
   - 터치 제스처 지원

### 레이아웃 개선
1. **사이드바**
   - 너비: 320px → 360-400px
   - 계층적 네비게이션
   - 접기/펼치기 애니메이션

2. **태스크 카드**
   - 높이 증가 (더 많은 정보 표시)
   - 미리보기 영역 추가
   - 상태 표시 강화

3. **메인 콘텐츠**
   - 최대 너비 제한 (읽기 편안함)
   - 그리드 시스템 활용
   - 공백 활용 증가

## 구현 우선순위

### Phase 1: 기초 개선 (1주)
1. 글자 크기 전체 조정
2. 패딩/마진 증가
3. 버튼 크기 증가
4. 기본 색상 팔레트 개선

### Phase 2: 시각적 개선 (2주)
1. 카드 디자인 개선
2. 그림자/효과 추가
3. 애니메이션 추가
4. 아이콘 시스템 정비

### Phase 3: 인터랙션 개선 (2주)
1. 드래그 앤 드롭 개선
2. 키보드 단축키
3. 마이크로 인터랙션
4. 반응형 디자인

### Phase 4: 고급 기능 (1주)
1. 테마 커스터마이징
2. 접근성 개선
3. 성능 최적화
4. 사용자 설정 저장
